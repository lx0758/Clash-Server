## ADDED Requirements

### Requirement: 导航栏流量指示器显示
系统 SHALL 在移动端导航栏显示实时流量指示器。

#### Scenario: 导航栏流量指示器布局
- **WHEN** 屏幕宽度小于 992px（isMobile = true）
- **THEN** 系统在移动端 Header（el-header v-if="isMobile"）显示流量指示器
- **AND** 流量指示器位于标题和用户菜单之间（header-right 区域）
- **AND** 使用 flex 布局，与用户菜单保持 12px 间距

#### Scenario: 流量指示器内容
- **WHEN** 流量指示器显示
- **THEN** 系统显示两个流量项：上行速度和下行速度
- **AND** 上行速度项包含：向上箭头图标（Top）、数值、单位
- **AND** 下行速度项包含：向下箭头图标（Bottom）、数值、单位
- **AND** 上行速度使用橙色（#f59e0b）
- **AND** 下行速度使用蓝色（#3b82f6）

#### Scenario: 流量指示器数据格式
- **WHEN** 流量指示器显示流量数据
- **THEN** 系统使用简化的单位格式（MB/s → M，KB/s → K）
- **AND** 数值保留 1 位小数（如 2.5M）
- **AND** 示例："↑2.5M"、"↓5.8M"

#### Scenario: 流量指示器实时更新
- **WHEN** WebSocket 推送新的 traffic 数据
- **THEN** 流量指示器立即更新上行和下行速度
- **AND** 更新频率与 WebSocket 推送频率一致（每秒一次）
- **AND** 更新时不引起布局抖动（使用固定宽度的数字）

#### Scenario: 流量指示器样式
- **WHEN** 流量指示器显示
- **THEN** 系统使用浅灰色背景（#f1f5f9）
- **AND** 使用圆角边框（border-radius: 4px）
- **AND** 内边距为 4px 8px（紧凑型）
- **AND** 字体大小为 12px（小号字体）
- **AND** 两个流量项之间的间距为 12px

#### Scenario: 流量指示器悬停效果
- **WHEN** 用户悬停在流量指示器上
- **THEN** 背景色加深（从 #f1f5f9 到 #e2e8f0）
- **AND** 光标变为指针样式
- **AND** 过渡动画时长为 200ms

### Requirement: 导航栏流量指示器交互
系统 SHALL 提供流量指示器的交互功能。

#### Scenario: 点击流量指示器跳转
- **WHEN** 用户点击流量指示器
- **THEN** 系统跳转到仪表盘页面（路由 "/"）
- **AND** 仪表盘页面显示完整的流量图表和统计信息
- **AND** 用户可以在仪表盘页面查看详细的流量趋势

#### Scenario: 流量指示器显示详细信息（可选）
- **WHEN** 用户悬停在流量指示器上（或长按触摸设备）
- **THEN** 系统显示详细信息弹窗（el-popover）
- **AND** 弹窗内容包括：当前速度、峰值速度、总流量、连接数
- **AND** 弹窗样式与流量指示器对齐

#### Scenario: 流量指示器无数据显示
- **WHEN** WebSocket 未连接或数据未推送
- **THEN** 系统显示加载状态（旋转图标）
- **AND** 或显示 "N/A" 文本
- **AND** 样式保持一致（灰色、圆角、边框）

### Requirement: 导航栏流量指示器响应式
系统 SHALL 确保流量指示器在不同移动设备上正常显示。

#### Scenario: 小屏移动设备适配
- **WHEN** 屏幕宽度小于 360px（如 iPhone SE）
- **THEN** 系统进一步缩小流量指示器的字体大小（从 12px 减到 11px）
- **AND** 减小内边距（从 4px 8px 减到 2px 6px）
- **AND** 减小流量项之间的间距（从 12px 减到 8px）

#### Scenario: 横屏模式适配
- **WHEN** 用户在移动设备上旋转到横屏
- **THEN** 系统重新计算流量指示器的布局
- **AND** 保持内容完整（不隐藏或截断）
- **AND** 调整字体和间距以适应横屏宽度

#### Scenario: 导航栏空间不足处理
- **WHEN** 导航栏宽度不足以同时显示标题、流量指示器和用户菜单
- **THEN** 系统隐藏用户名，只显示用户头像图标
- **AND** 流量指示器保持显示（核心功能）
- **AND** 用户菜单通过点击头像图标展开

### Requirement: 导航栏流量指示器性能优化
系统 SHALL 确保流量指示器不影响移动端性能。

#### Scenario: 流量指示器轻量级渲染
- **WHEN** 流量指示器更新
- **THEN** 系统只更新文本内容（不重新渲染整个组件）
- **AND** 使用 Vue 的响应式系统（ref、computed）
- **AND** 避免不必要的 DOM 操作

#### Scenario: 流量指示器节流更新
- **WHEN** WebSocket 推送频率过高（如每秒多次）
- **THEN** 系统限制流量指示器的更新频率（最多每秒 1 次）
- **AND** 使用 requestAnimationFrame 优化更新时机
- **AND** 避免过度的 UI 更新

#### Scenario: 流量指示器卸载时清理
- **WHEN** 用户离开移动端页面（如跳转到登录页）
- **THEN** 系统取消 WebSocket 订阅（如果其他页面不需要流量数据）
- **AND** 清理流量指示器的定时器和事件监听器
- **AND** 释放内存资源

### Requirement: 导航栏流量指示器可访问性
系统 SHALL 确保流量指示器易于访问和使用。

#### Scenario: 流量指示器键盘导航
- **WHEN** 用户使用键盘 Tab 键导航
- **THEN** 流量指示器可获得焦点（tabindex="0"）
- **AND** 按 Enter 或 Space 键触发点击事件（跳转到仪表盘）
- **AND** 焦点显示清晰的外框（outline）

#### Scenario: 流量指示器屏幕阅读器支持
- **WHEN** 屏幕阅读器读取导航栏
- **THEN** 流量指示器包含 aria-label 属性
- **AND** aria-label 内容："实时流量：上行 [上行速度]，下行 [下行速度]"
- **AND** aria-live 属性设置为 "polite"（实时更新时通知屏幕阅读器）

#### Scenario: 流量指示器高对比度模式
- **WHEN** 用户启用高对比度模式
- **THEN** 系统调整流量指示器的颜色（橙色/蓝色 -> 更亮的橙色/蓝色）
- **AND** 增加文字和图标的对比度
- **AND** 保持内容的可读性

### Requirement: 导航栏流量指示器仅在移动端显示
系统 SHALL 确保流量指示器仅在移动端导航栏显示。

#### Scenario: 桌面端不显示流量指示器
- **WHEN** 屏幕宽度大于等于 992px（isMobile = false）
- **THEN** 系统不渲染流量指示器组件
- **AND** 桌面端 Header 不包含流量指示器（使用侧边栏导航）
- **AND** 用户在仪表盘页面查看完整的流量图表和统计

#### Scenario: 断点切换时流量指示器显示/隐藏
- **WHEN** 用户调整窗口大小，断点在移动端和桌面端之间切换
- **THEN** 系统根据新的断点显示或隐藏流量指示器
- **AND** 切换时保持平滑过渡（无闪烁或布局跳变）
- **AND** 状态和数据保持一致（不丢失 WebSocket 连接）
