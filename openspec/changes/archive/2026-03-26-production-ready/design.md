## Context

当前项目使用外部静态文件目录 `server/web/` 存放 Vue 构建产物，部署时需要同时携带二进制和静态资源文件。Session 管理使用 `gorilla/sessions`，需要手动初始化 CookieStore。

此外，发现若干 UI 问题：
- PC 端侧边栏缺少登出入口，用户无法退出登录
- 仪表盘核心状态显示冗余（"运行中 · v1.0.0"）
- 激活订阅的绿色边框样式过于突兀
- 小屏幕规则编辑器删除按钮占用过多空间

### 约束条件

- Go embed 不支持 `..` 路径，embed 文件必须与资源目录同级或在父级
- gin-contrib/sessions 需要在路由初始化时配置 store
- Element Plus 组件样式覆盖需使用 `:deep()` 选择器

## Goals / Non-Goals

**Goals:**

- 单一二进制文件部署，无需外部静态资源依赖
- 使用社区标准中间件，减少手写代码
- 简化模块导入路径
- 保持现有功能不变
- PC 端提供登出入口
- 优化 UI 视觉表现

**Non-Goals:**

- 不改变 API 接口
- 不改变前端构建流程（仅调整输出路径）
- 不引入数据库 session 存储
- 不重新设计整体 UI 风格

## Decisions

### D1: 静态资源嵌入方案

**选择**: Go `embed.FS` + `fs.Sub`

**理由**: 
- Go 1.16+ 原生支持，无需第三方依赖
- 编译时嵌入，运行时无 IO 开销
- 支持 SPA 路由回退（所有 404 返回 index.html）

**目录结构**:
```
server/res/
├── res.go           # embed 定义
└── web/             # Vue 产物
    ├── index.html
    └── assets/
```

**替代方案**:
- `packr` / `statik`: 需要额外构建步骤，已过时
- 外部文件: 部署复杂度高

### D2: Session 管理方案

**选择**: `gin-contrib/sessions` + `memstore`

**理由**:
- 与 Gin 框架原生集成，通过 `sessions.Default(c)` 访问
- 无需全局变量和手动初始化
- 支持 memstore/redis/cookie 等多种后端

**API 变化**:
```go
// 之前
session, _ := middleware.Store.Get(c.Request, middleware.SessionName)
session.Values["user_id"] = user.ID
session.Save(c.Request, c.Writer)

// 之后
middleware.LoginUser(c, user.ID)
```

**替代方案**:
- 继续使用 gorilla/sessions: 可行但代码冗余
- JWT 无状态: 需要更大改动，不符合当前需求

### D3: CORS 中间件方案

**选择**: `gin-contrib/cors.Default()`

**理由**:
- 一行配置替代 11 行手写代码
- 自动处理预检请求
- 社区维护，安全更新及时

### D4: PC 端登出入口设计

### D5: 核心状态显示优化

**选择**: 运行时只显示版本号，停止时显示"已停止"

**显示逻辑**:
```
运行中: v1.18.0
已停止: 已停止
```

**理由**:
- 运行状态由绿色图标已表达，无需文字重复
- 版本号是用户更关心的信息

### D6: 激活订阅样式优化

**选择**: 移除边框，改用左侧竖线指示

**样式变化**:
```css
/* 之前 */
.subscription-card.active {
  border-color: #10b981;
  box-shadow: 0 0 0 1px #10b981;  /* 移除 */
}

/* 之后 */
.subscription-card.active {
  border-left: 3px solid #10b981;  /* 添加 */
}
```

**理由**:
- 左侧竖线更柔和，不喧宾夺主
- 保持激活状态的视觉区分

### D8: 连接信息时间显示优化

**选择**: 添加连接开始时间显示在主机名前，统一格式 `HH:mm:ss.SSS`

**布局设计**:
```
┌──────────────────────────────────────────────────────────────┐
│  大屏布局                                                    │
├──────────────────────────────────────────────────────────────┤
│  22:01:03.234 - 主机名 [已断开]                            │
│  [TCP / HTTP / 目标:端口]              ↑ 1.2 KB ↓ 5.6 KB   │
│  [rule => chain]                          [关闭]            │
└──────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────┐
│  小屏布局 (<640px)                                           │
├──────────────────────────────────────────────────────────────┤
│  22:01:03.234 - 主机名                                      │
│  [TCP / HTTP / 目标:端口]                                   │
│  [rule => chain]                                             │
│  ↑ 1.2 KB ↓ 5.6 KB                          [关闭]          │
└──────────────────────────────────────────────────────────────┘
```

**实现方式**:
- 时间显示在主机名前面，中间用 ` - ` 分隔
- 数据源: `conn.start` (ISO 时间字符串)
- 格式化函数: 提取时分秒毫秒部分
- 时间和主机名使用相同字体大小，颜色区分

### D10: 连接数统计优化

**选择**: 仅显示激活的连接数，使用 Link 图标

**实现方式**:
- 标签内显示 `{{ activeCount }} 个`
- 图标使用 Element Plus 的 `Link` 图标
- 颜色使用 `type="warning"` (橙色)

### D9: 日志格式优化

**选择**: 统一时间格式 `HH:mm:ss.SSS`，小屏采用两行布局（时间和 level 在第一行，payload 在第二行）

**布局设计**:
```
┌──────────────────────────────────────────────────────────────┐
│  大屏布局                                                    │
├──────────────────────────────────────────────────────────────┤
│  [22:01:03.234] [INFO] 用户登录成功                         │
│  [22:01:05.123] [WARN] 连接超时，重试中                     │
│  [22:02:00.456] [ERROR] 无法连接到服务器                    │
└──────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────┐
│  小屏布局 (<640px)                                           │
├──────────────────────────────────────────────────────────────┤
│  [22:01:03.234] [INFO]                                      │
│  用户登录成功                                               │
│                                                              │
│  [22:01:05.123] [WARN]                                      │
│  连接超时，重试中                                            │
└──────────────────────────────────────────────────────────────┘
```

**实现方式**:
- 时间格式: 去掉日期，只显示 `HH:mm:ss.SSS`
- 大屏: 时间、level 和 payload 在同一行
- 小屏: 时间和 level 在第一行，payload 单独一行
- 保持方框样式和颜色编码

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| 二进制体积增大 (~500KB) | 可接受，换取部署简化 |
| 静态资源更新需重新编译 | 开发环境使用 `npm run dev` 直连 |
| memstore 重启丢失 session | 单实例部署可接受，未来可迁移到 redis |
| 左侧竖线在折叠状态不可见 | 侧边栏展开时可见，可接受 |

## Migration Plan

### 步骤

1. 创建 `server/res/` 目录，移动 web 资源
2. 创建 `res.go` 定义 embed
3. 更新 `main.go` 路由配置
4. 更新所有导入路径
5. 添加新依赖，更新 go.mod
6. 更新 `.gitignore` 忽略构建产物
7. 更新前端组件 (MainLayout, RuntimeInfoCard, SubscriptionCard, RuleEditorDialog)

### 回滚策略

- git revert 即可恢复到之前状态
- 无数据库迁移，无风险
