## Context

当前系统启动和配置管理流程：

```
启动: mihomo -f temp/config-xxx.yaml
变更: 杀进程 → 重新启动 mihomo -f temp/config-yyy.yaml
```

问题：
- 启动时需要等待配置加载
- 变更需要杀进程重启
- 连接中断，用户体验差
- 需要管理临时配置文件

Mihomo 提供的 API 能力：
```
PUT  /configs?force=true  → 热重载配置
POST /restart              → 软重启（不杀进程）
```

## Goals / Non-Goals

**Goals:**
- Mihomo 启动时使用最小配置，启动后通过 API 应用全量配置
- 所有配置变更统一通过 Mihomo API 管理
- 热重载优先，软重启兜底
- 移除临时配置文件机制

**Non-Goals:**
- 不改变现有业务 API 接口签名
- 不实现前端 UI 变更（仅后端改造）

## Decisions

### 决策 1：启动流程重构

**选择**: Mihomo 启动时通过 `-config` 参数传入 Base64 编码的最小配置，启动后通过 API 应用全量配置

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         新启动流程                                          │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   1. 从 CoreConfig 获取 API 监听配置                                        │
│      cfg := config.GetCoreConfig()                                         │
│      apiAddr := fmt.Sprintf("%s:%d", cfg.APIHost, cfg.APIPort)             │
│                                                                             │
│   2. 生成最小配置并 Base64 编码                                             │
│      配置内容: "external-controller: 127.0.0.1:9090" (动态生成)            │
│      Base64: 动态计算                                                       │
│                                                                             │
│   3. 启动 Mihomo                                                           │
│      mihomo -d /path/to/workdir -config <base64>                           │
│      → 监听配置的 APIHost:APIPort，无 secret                               │
│                                                                             │
│   4. 等待 API 就绪                                                         │
│                                                                             │
│   5. 调用 PUT /configs 应用全量配置                                        │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**理由**:
- 启动更快（最小配置加载）
- 统一通过 API 管理配置
- 不再需要临时配置文件
- `-config` 参数直接传入 Base64 编码配置，无需文件
- API 端口从配置获取，确保业务能正确访问

### 决策 2：配置变更统一通过 API

**选择**: 所有配置变更通过 Mihomo RESTful API

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         配置变更流程                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   Handler.applyConfig() → CoreService.ApplyConfig() → 智能判断             │
│                                                        │                    │
│                     ┌────────────────────────────────┴──────────────┐       │
│                     ▼                                             ▼       │
│              HotReload()                                   SoftRestart()  │
│              (PUT /configs)                                (POST /restart)│
│                                                                             │
│   热重载失败 → 返回 need_confirm → 用户确认 → SoftRestart()               │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**理由**:
- API 方式更可控
- 软重启比杀进程更稳定
- 移除临时文件管理复杂度

### 决策 3：需要软重启的配置项

以下配置项变更需要调用 POST /restart：

| 配置项 | 说明 |
|--------|------|
| mixed-port | 混合代理端口 |
| external-controller | API 监听地址 |
| secret | API 密钥 |
| bind-address | 绑定地址 |
| tun | TUN 模式 |
| interface-name | 出站接口 |
| routing-mark | 路由标记 |
| ipv6 | IPv6 开关 |

其他配置项（proxies, rules, dns 等）可通过 PUT /configs 热重载。

### 决策 4：热重载失败处理

**选择**: 返回 need_confirm，前端询问用户是否软重启

**理由**:
- 用户有选择权
- 软重启比热重载影响大，需要用户确认

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|---------|
| API 不可用时无法启动 | 启动时检测 API 就绪，超时返回错误 |
| 软重启失败 | 捕获错误，降级为硬重启（杀进程） |
| 配置过大导致 API 超时 | 增加超时时间，或考虑配置压缩 |
| Base64 编码配置长度限制 | 最小配置很短，不会触发限制 |
