# 详细设计

## 1. 数据模型

### 1.1 Subscription 模型变更

```
┌─────────────────────────────────────────────────────────────────────┐
│  Subscription                                                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  字段变更:                                                           │
│  ┌────────────────┬─────────────┬─────────────────────────────────┐ │
│  │ 字段            │ 类型        │ 说明                            │ │
│  ├────────────────┼─────────────┼─────────────────────────────────┤ │
│  │ id             │ uint        │ 主键                            │ │
│  │ name           │ string      │ 名称                            │ │
│  │ source_type    │ string      │ "remote" | "local"              │ │
│  │ url            │ string?     │ 远程订阅 URL (local 时为空)      │ │
│  │ interval       │ int?        │ 刷新间隔分钟 (local 时为空)      │ │
│  │ content        │ text        │ 订阅内容 YAML                   │ │
│  │ active         │ bool        │ 是否激活                        │ │
│  │ use_proxy      │ bool        │ 是否使用代理更新订阅             │ │
│  │ proxy_name     │ string?     │ 使用的代理名称 (空=系统代理)     │ │
│  │ user_agent     │ string?     │ 自定义 User-Agent               │ │
│  │ skip_cert      │ bool        │ 是否跳过证书校验                │ │
│  │ last_refresh   │ time?       │ 上次刷新时间                    │ │
│  │ node_count     │ int         │ 节点数量                        │ │
│  │ upload_used    │ int64       │ 已用上传流量 (字节)              │ │
│  │ download_used  │ int64       │ 已用下载流量 (字节)              │ │
│  │ total_transfer │ int64       │ 总流量 (字节)                    │ │
│  │ expire_at      │ time?       │ 订阅过期时间                    │ │
│  │ created_at     │ time        │ 创建时间                        │ │
│  │ updated_at     │ time        │ 更新时间                        │ │
│  └────────────────┴─────────────┴─────────────────────────────────┘ │
│                                                                      │
│  约束:                                                               │
│  • source_type = "remote" 时, url 必填                              │
│  • source_type = "local" 时, url/interval/use_proxy/skip_cert 为空  │
│  • 同一时间只有一个 active = true                                    │
│                                                                      │
│  说明:                                                               │
│  • use_proxy=true 且 proxy_name 为空: 使用系统代理                  │
│  • use_proxy=true 且 proxy_name 有值: 使用指定代理节点              │
│  • user_agent 为空时使用默认 UA                                     │
│  • skip_cert=true 用于某些机场证书问题                              │
│  • upload_used/download_used/total_transfer 来自 subscription-userinfo 响应头 │
│  • expire_at 来自 subscription-userinfo 中的 expire 字段 (Unix 时间戳) │
│  • 流量字段为 0 表示机场未提供此信息                                  │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.5 subscription-userinfo 解析

```
┌─────────────────────────────────────────────────────────────────────┐
│  subscription-userinfo 响应头解析                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  原始响应头:                                                         │
│  subscription-userinfo: upload=240935845; download=15758204665;     │
│                        total=1099511627776; expire=1796263142       │
│                                                                      │
│  解析规则:                                                           │
│  ┌────────────┬──────────────────────────────────────────────────┐  │
│  │ 字段        │ 说明                                             │  │
│  ├────────────┼──────────────────────────────────────────────────┤  │
│  │ upload     │ 已上传流量 (字节) → upload_used                  │  │
│  │ download   │ 已下载流量 (字节) → download_used                │  │
│  │ total      │ 总流量 (字节) → total_transfer                   │  │
│  │ expire     │ 过期时间 (Unix 时间戳) → expire_at               │  │

│  • 部分机场可能不返回此头，此时字段保持为 0/null                     │
│  • expire 为 0 或负数时，不设置 expire_at                           │
│  • 刷新订阅时解析并更新这些字段                                      │
│                                                                      │

```

## 3. 配置合并逻辑变更

### 3.1 当前合并流程

```
┌─────────────────────────────────────────────────────────────────────┐
│  当前 Merger.Merge()                                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. 获取激活订阅的 content                                           │
│  2. 解析为 baseConfig                                               │
│  3. 获取全局 insertRules, appendRules                               │
│  4. 应用 insertRules 到 baseConfig                                  │
│  5. 应用 coreConfig                                                 │
│  6. 应用 appendRules 到 baseConfig                                  │
│  7. 执行所有启用的全局脚本                                           │
│  8. 返回最终配置                                                     │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 3.2 新合并流程

```
┌─────────────────────────────────────────────────────────────────────┐
│  新 Merger.Merge()                                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  1. 获取激活订阅 (包含 content)                                      │
│  2. 解析为 baseConfig                                               │
│  3. 获取该订阅关联的 insertRules, appendRules                       │
│  4. 应用 insertRules 到 baseConfig                                  │
│  5. 应用 coreConfig                                                 │
│  6. 应用 appendRules 到 baseConfig                                  │
│  7. 执行该订阅关联的启用脚本                                         │
│  8. 返回最终配置                                                     │
│                                                                      │
│  关键变化:                                                           │
│  • 规则和脚本按 subscription_id 过滤                                 │
│  • 切换订阅自动切换对应的规则/脚本                                   │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## 5. 数据迁移策略

### 5.1 迁移方案

```
┌─────────────────────────────────────────────────────────────────────┐
│  迁移策略: 创建默认本地配置                                          │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  步骤:                                                               │
│                                                                      │
│  1. 添加新字段 (subscription.source_type,                           │
│                 rule.subscription_id,                               │
│                 script.subscription_id)                              │
│                                                                      │
│  2. 更新现有订阅:                                                    │
│     UPDATE subscriptions SET source_type = 'remote';                │
│                                                                      │
│  3. 创建默认本地配置:                                                │
│     INSERT INTO subscriptions (name, source_type, active, content)  │
│     VALUES ('默认配置', 'local', false, '{}');                      │
│                                                                      │
│  4. 关联现有规则和脚本到默认本地配置:                                 │
│     UPDATE rules SET subscription_id = <默认配置ID>;                │
│     UPDATE scripts SET subscription_id = <默认配置ID>;              │
│                                                                      │
│  5. 如果有激活订阅:                                                  │
│     将其规则/脚本迁移到该激活订阅                                    │
│     (用户可能更希望这样)                                             │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 5.2 迁移 SQL

```sql
-- Step 1: 添加新字段
ALTER TABLE subscriptions ADD COLUMN source_type VARCHAR(16) DEFAULT 'remote';
ALTER TABLE rules ADD COLUMN subscription_id UINT;
ALTER TABLE scripts ADD COLUMN subscription_id UINT;

-- Step 2: 设置现有订阅为远程类型
UPDATE subscriptions SET source_type = 'remote' WHERE source_type IS NULL;

-- Step 3: 获取或创建默认配置
-- (在应用层处理，确保正确关联)

-- Step 4: 创建索引
CREATE INDEX idx_rules_subscription_id ON rules(subscription_id);
CREATE INDEX idx_scripts_subscription_id ON scripts(subscription_id);

-- Step 5: 添加外键约束
ALTER TABLE rules ADD CONSTRAINT fk_rules_subscription 
  FOREIGN KEY (subscription_id) REFERENCES subscriptions(id) ON DELETE CASCADE;
ALTER TABLE scripts ADD CONSTRAINT fk_scripts_subscription 
  FOREIGN KEY (subscription_id) REFERENCES subscriptions(id) ON DELETE CASCADE;
```
