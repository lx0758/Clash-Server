# 提案：订阅关联规则与脚本 + 本地配置

## 背景

当前架构中，自定义规则和脚本是全局的，与订阅无关。这导致：

1. 切换订阅后，规则和脚本保持不变，可能产生不一致
2. 无法针对不同机场使用不同的规则策略
3. 无法创建纯本地配置（所有订阅必须有 URL）

## 目标

1. 将规则和脚本关联到具体订阅
2. 引入"本地配置"概念，支持无 URL 的订阅
3. 保持数据迁移平滑

## 变更范围

```
┌─────────────────────────────────────────────────────────────────────┐
│                        影响范围                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  后端:                                                              │
│  ├── 数据模型: Subscription, Rule, Script                         │
│  ├── 服务层: SubscriptionService, RuleService, ScriptService      │
│  ├── 处理器: RuleHandler, ScriptHandler, SubscriptionHandler      │
│  ├── 配置合并: Merger                                              │
│  └── 数据迁移: 新增迁移脚本                                         │
│                                                                     │
│  前端:                                                              │
│  ├── 页面: Subscriptions.vue (重构)                                │
│  ├── API: rule.ts, script.ts (路径变更)                            │
│  └── 类型: api.ts                                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 非目标

- 不改变规则/脚本的数据结构本身
- 不改变配置合并的基本逻辑顺序
- 不引入多订阅同时激活
